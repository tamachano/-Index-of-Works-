## イタリア名所クイズ
http://italy-quiz-next.s3-website-us-east-1.amazonaws.com

### 作成理由
正直な所、初めはReactコンポーネントの理解を含めたクイズアプリを作成することが目的でした。  

しかし、実際に作成する中で他の作品との結びつきを考えたり、  
「面白いことは何かできないだろうか。」  
「より見てくれた人に楽しんでもらえないだろうか？」  
という想いが出てくる中で、今のクイズアプリが生まれました。

### 現状分析
クイズを進める中で、問題文はすぐに表示される一方、
画像の読み込みに時間がかかる場面が多く、ユーザーの視線や思考が分断されてしまうと感じました。
その結果、テンポが崩れ、離脱につながりやすい状態になっていました。

### 改善点
読み込み体験を改善するため、
Next.jsのpreloadや画像最適化を用いて、
問題文と視覚情報が同時に届く体験を目指したいと考えています。

## Aurora
https://d17sydrv5aw49e.cloudfront.net/

### 作成理由
私がビジターだったら、  
「こんなサイトであれば魅力的でイタリアに行きたいと感じるであろう」  
という気持ちを詰めました。  

画像の配置が一定にならないことでユーザーを飽きさせない工夫や、  
UIをモダンなものにすることで高級感を向上させています。

### 改善点
⓵mainvisualが大きな画像のため、ロードに時間が取られUXが低下しやすい。  

文字を先に見せて「何のサイトか」を先に認知してもらうことで、  
ストレスが下がりUXが良くなる可能性があると考えました。  

ただし、mainvisualスライドのpreloadを行うと  
全体の読み込みが遅くなる恐れがあったため断念しています。

## Auronox
現在一時閉鎖中

### 作成理由
Auroraは夜明け後の外の世界、
Auronoxは夜（nox）の内側を描写した没入体験として作成しました。  

文言ですべてを説明しきらず、  
解釈の【余白】を残すことでユーザーの心地よさを向上させる狙いがあります。  

クイズを進めることで徐々に周囲が明るくなり、Auroraへとつながる裏方的な位置づけになりました。

### 改善点
世界観の重視は良かったものの、  
入り口としてクイズアプリと並列に設けるべきでした。  

また、最初のページ中央リンクからの遷移は  
UIとして不親切であった可能性があります。  

加えて、画像読み込み時間も課題です。

## それ、買うbuy？
https://dmvig8face73p.cloudfront.net/
買い物の「迷い」を一瞬で軽くする、簡易プロダクトを作りました。(1/13)
### 作成理由
私自身、買い物や自己投資の機会が多く、
『気分が上がるか』『今の自分に合っているか』を基準に選ぶことが多い一方で、
あとから「本当に今必要だったのか？」と感じることも少なくありませんでした。

そうした経験から、
買う・買わないを決める前に、『自分の状態』を軽く確認できる
シンプルなセルフチェックアプリがあればいいのではないかと考え、本プロダクトを制作しました。

あえて機能や質問を最小限に絞り、どこまで削ぎ落とせば、価値が残るのか
という点も一つのテーマとしています。

このプロダクトは、生活の中の判断コストを下げるための小さな道具として設計しています。
