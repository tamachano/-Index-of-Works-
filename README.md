## イタリア名所クイズ  
### 作成理由
正直な所、初めはReactコンポーネントの理解を含めたクイズアプリを作成することが私の目的でした。
しかし、クイズアプリを実際に作成する中で他の作品との結びつきを考えたり、

「面白いことは何かできないだろうか。」  
「より見てくれた人に楽しんでもらえないだろうか？」  

という想いが出てくる中で今のクイズアプリが生まれました。
また、クイズのUIとして50:50機能などの実装も考えましたが、テーマにそぐわないため、実装せず。

### 改善点  
画像の読み込みが遅く、UXの低下を招いて離脱が起きやすい。  
→ Next.jsのpreloadなどで、読み込み体験の改善が必要

●Aurora  
【作成理由】    
私がビジターだったら、  
『こんなサイトであれば魅力的でイタリアに行きたいと感じるであろう』  
という気持ちを詰めました。 

画像の配置が一定にならないことでユーザーを飽きさせないという工夫や、
UIをモダンなものにすることにより高級感を向上させています。

### 改善点  
mainvisualが大きな画像の為、ロードに時間がとられUXが低下しやすい。  
⓵文字を先に見せて「何のサイトか」を先に認知してもらうことで、  
ストレスが下がりUXが良くなる可能性があるため、mainvisualに画像のみを採用するのではなく、
文字との併用によりUXの向上につなげる。
単なるmainvisualのslideのpreloadを実行すると、全体の読込が遅くなる恐れがあるため、断念。

## Auronox   
### 作成理由      
Auroraは夜明けした外の世界に対し、AuronoxはAurora（夜明け・暁）が見えない、nox（夜）の内側を強く描写した没入体験として作成しました。  

文言として説明しきらない。完全に語りきらないことで解釈の【余白】を残すことで、ユーザーの心地よさを向上させるという狙いがあります。  

クイズを解いていくと気づかないうちに周りが明るくなり、Auroraへつながる裏方としての位置付けとしての意味合いが強くなりました。

### 改善点  
世界観の重視は良かったが、入り口として上記クイズアプリと並列で設けるべきでした。
最初のページ中央リンクからの遷移はＵＩとして不親切であった可能性もあります。

また、前述のクイズアプリ同様、画像の読み込み時間もあります。
